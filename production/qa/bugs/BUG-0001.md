## Technical Context

### Root Cause
`sts2.csproj` 中 DLL 引用的 HintPath 指向不存在的路径：

| DLL | 错误路径 | 正确路径 |
|-----|----------|----------|
| Steamworks.NET | `..\..\Games\Slay the Spire 2\data_sts2_windows_x86_64\` | `.godot\mono\temp\bin\Debug\` |
| 0Harmony | `..\..\Games\Slay the Spire 2\data_sts2_windows_x86_64\` | `.godot\mono\temp\bin\Debug\` |

### Affected Files (17+)
- `src/Core/ControllerInput/SteamControllerInputStrategy.cs`
- `src/Core/Modding/ModManager.cs`
- `src/Core/Platform/Steam/*.cs`
- `src/Core/Multiplayer/Transport/Steam/*.cs`
- `src/Core/Saves/SaveManager.cs`
- `src/Core/Entities/Multiplayer/NetErrorInfo.cs`

## Fix Applied

**Date**: 2026-05-15
**Fixed by**: Claude Code

修改 `sts2.csproj` 第 25-30 行：

```xml
<!-- Before -->
<Reference Include="Steamworks.NET">
  <HintPath>..\..\Games\Slay the Spire 2\data_sts2_windows_x86_64\Steamworks.NET.dll</HintPath>
</Reference>
<Reference Include="0Harmony">
  <HintPath>..\..\Games\Slay the Spire 2\data_sts2_windows_x86_64\0Harmony.dll</HintPath>
</Reference>

<!-- After -->
<Reference Include="Steamworks.NET">
  <HintPath>.godot\mono\temp\bin\Debug\Steamworks.NET.dll</HintPath>
</Reference>
<Reference Include="0Harmony">
  <HintPath>.godot\mono\temp\bin\Debug\0Harmony.dll</HintPath>
</Reference>
```
